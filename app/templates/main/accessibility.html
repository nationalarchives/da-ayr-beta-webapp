    Accessibility statement for Access Your Records

    This accessibility statement applies to https://ayr.nationalarchives.gov.uk

    This website is run by The National Archives. We want as many people as possible to be able to use this website. For example, that means you should be able to:

    • change colours, contrast levels and fonts using browser or device settings
    • zoom in up to 400% without the text spilling off the screen
    • navigate most of the website using a keyboard or speech recognition software
    • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

    We've also made the website text as simple as possible to understand.

    AbilityNet has advice on making your device easier to use if you have a disability.

    Technical information about this website's accessibility

    The National Archives is committed to making its website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

    Compliance status

    This website is currently compliant with the Web Content Accessibility Guidelines version 2.2 AA standard.

    Non-accessible content

    As of the 31st of July 2025 the website does not contain non-compliant content.

    Resolved accessibility issues

    The following accessibility issues have been resolved in line with WCAG 2.2 Level A and AA requirements:

    • Breadcrumbs have been moved to a more appropriate section of the page in order to prevent focus issues.
    • Static pages have been adjusted to contain correct heading levels and consistent layout.
    • Radio buttons under "Search for digital records" now have accessible names that match their visible labels, enabling proper voice control interaction.
    • Pages now have a single <h1> heading to reflect the main topic, with correct use of <h2> and <legend> for structure clarity in screen readers.
    • Additional content shown in expandable rows within tables is now programatically associated with the related table data.
    • Expand/collapse controls in search results now include descriptive accessible names.
    • "Remove filter" buttons no longer nest anchor tags, ensuring predictable and accessible interaction for all users.
    • PDF viewer navigation controls now use semantic HTML and appropriate ARIA roles, additionally, the "Go" button that is part of the controls cluster is no longer hidden behind content when focused.
    • Status updates like "No results found" are now coded with ARIA live controls, so they are announced to screen readers.
    • Layout adjusts correctly at various different resolutions and zoom levels.
    • Some buttons have been given more accessible and descriptive names, such as buttons that contain "Next" being given an ARIA label of "Next page".

    Preparation of this accessibility statement

    This website was last tested on 21st March 2025.

    The test was carried out by Digital Accessibility Centre. The most viewed pages were tested using automated testing tools by our website team. A further audit of the website was carried out to the WCAG 2.2 AA standard.
